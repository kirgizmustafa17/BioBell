# Plan 01-03: Hilt DI Setup

## Objective

Set up Hilt as the DI framework with Application class, entry points, and an empty AppModule scaffold.

## Tasks

### Task 1: BioBellApplication.kt

```kotlin
package com.biobell.android

import android.app.Application
import dagger.hilt.android.HiltAndroidApp

@HiltAndroidApp
class BioBellApplication : Application()
```

### Task 2: AppModule.kt (empty scaffold)

```kotlin
package com.biobell.android.di

import dagger.Module
import dagger.hilt.InstallIn
import dagger.hilt.components.SingletonComponent

@Module
@InstallIn(SingletonComponent::class)
object AppModule {
    // Dependencies will be provided here in future phases
}
```

### Task 3: MainActivity.kt (initial — just the Hilt entry point, NavHost comes in Plan 01-04)

```kotlin
package com.biobell.android

import android.os.Bundle
import androidx.activity.ComponentActivity
import androidx.activity.compose.setContent
import androidx.activity.enableEdgeToEdge
import com.biobell.android.ui.theme.BioBellTheme
import dagger.hilt.android.AndroidEntryPoint

@AndroidEntryPoint
class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        setContent {
            BioBellTheme {
                // NavHost will be wired in Plan 01-04
            }
        }
    }
}
```

## Must-Haves

- [ ] `BioBellApplication` annotated with `@HiltAndroidApp`
- [ ] `MainActivity` annotated with `@AndroidEntryPoint`
- [ ] App compiles and launches without Hilt crash
- [ ] `AppModule` present in di package (even if empty)
