# Plan 01-05: Backup Rules & Smoke Test

## Objective

Configure Android Auto Backup rules and create CLAUDE.md with project conventions for future AI agents.

## Tasks

### Task 1: Backup configuration XMLs

`app/src/main/res/xml/backup_rules.xml` (Android 11 and below):
```xml
<?xml version="1.0" encoding="utf-8"?>
<full-backup-content>
    <!-- Include alarm database and DataStore preferences -->
    <include domain="database" path="biobell_alarms.db" />
    <include domain="sharedpref" path="." />
    <!-- Exclude cache and temp files -->
    <exclude domain="cache" path="." />
    <exclude domain="external-cache" path="." />
</full-backup-content>
```

`app/src/main/res/xml/data_extraction_rules.xml` (Android 12+):
```xml
<?xml version="1.0" encoding="utf-8"?>
<data-extraction-rules>
    <cloud-backup>
        <include domain="database" path="biobell_alarms.db" />
        <include domain="sharedpref" path="." />
        <exclude domain="cache" path="." />
    </cloud-backup>
    <device-transfer>
        <include domain="database" path="biobell_alarms.db" />
        <include domain="sharedpref" path="." />
    </device-transfer>
</data-extraction-rules>
```

```
## Critical Rules

### Alarm Reliability (DO NOT SKIP)
- ALWAYS use `AlarmManager.setAlarmClock()` for user-facing alarms — NEVER `setExact()` alone
- ALWAYS use `FLAG_IMMUTABLE` on all PendingIntents
- ALWAYS reschedule alarms in BootReceiver (BOOT_COMPLETED)
- Use `USE_EXACT_ALARM` permission (not `SCHEDULE_EXACT_ALARM`)

### Sleep Math
- Sleep cycle duration: 90 minutes
- Sleep onset offset: 15 minutes (always subtract from usable sleep time)
- Chronotype offsets: 🐓 Early Bird = -30 min, 🐦 Intermediate = 0, 🦉 Night Owl = +30 min
- Minimum healthy sleep: 6 hours (4 cycles)
- Use `LocalDateTime` for alarm math (NOT `LocalTime` — crosses midnight)

### Code Conventions
- All ViewModels annotated with `@HiltViewModel` and inject via `hiltViewModel()`
- Repositories are interfaces in domain/; implementations in data/
- Use `collectAsStateWithLifecycle()` not `collectAsState()` in Compose
- Commit each task atomically with conventional commit messages
- No hardcoded strings — use strings.xml resources

### Design
- Dark theme is default; always test both dark and light
- Use `MaterialTheme.colorScheme.*` semantic roles — never hardcode colors
- Primary accent: follows dynamic color (seed: #5B4FCF indigo fallback)
```
